**As your FIRST action, tell the human in chat, in one or two lines:** you are installing a local SEO skill (map pack rankings, Google Business Profile, citations, reviews, location pages) that you will apply across their future work; nothing is needed beyond writing these files, no accounts or keys; about two minutes plus one question. ## Install the files unchanged

1. Detect where this environment keeps reusable skills or instructions. If it supports a folder per skill (a skills directory), create ONE folder named `local-seo` and write both files into it preserving the exact layout: `SKILL.md` at the folder root, `kb-distilled.md` under `references/`. The split is deliberate: the method file covers most work, and the deeper reference loads only when the task needs tactics, diagnosis, or case-study depth, so the skill does not occupy context it does not need.
2. If this environment can hold only a single instruction blob, concatenate the files in this order into one document: `SKILL.md`, then `references/kb-distilled.md`. Concatenation loses nothing; the reference table at the end of `SKILL.md` then points at the sections below it.
3. If a skill or file named `local-seo` already exists here, do NOT overwrite it. Back it up beside itself first, then ask the human whether to replace it.
4. If this environment already carries a comparable local SEO skill or instruction set, STOP and reconcile with the human: extend the existing one, replace it, or keep both under clearly distinct names. Never leave two overlapping local SEO instruction sets silently steering the same answers.
5. Write nothing anywhere else.

## Calibrate (one question)

Ask the human ONE question via your interactive question UI, and persist the answer next to the skill:

> "Which shape is the business this skill will mostly serve? (a) Storefront customers visit, (b) Service-area business (you go to them), (c) Multi-location, (d) Still planning or just starting."

The files already branch on exactly this: storefront businesses show their address and optimize the physical-location signals; service-area businesses hide the address, set a service radius, and follow the service-area rules (including the anti-patterns about distant cities); multi-location businesses follow the one-GBP-per-location strategy and the city-hub URL architecture; a new business starts with the foundational sequence before anything advanced. Your answer decides which branches you read as primary from now on. The calibration is re-runnable; offer to re-run it when the human's focus appears to have changed, presenting the current value as the editable default.
